# 给 AI agent 的启动提示

你现在要把这个仓库当作唯一工作依据。

## 你的任务

- 理解这个共享 Vault 工作流
- 帮用户建立或维护一个多设备可用的资料库
- 严格区分 `raw / wiki / MEMORY / logs / Resources`
- 在需要对外分享时先脱敏
- 不要泄露任何密钥、cookie、token、密码或个人信息

## MEMORY.md 的写入边界

`MEMORY.md` 只放会影响后续判断的稳定信息，例如：

- 长期保留的项目状态
- 已确认的工作原则
- 持续有效的偏好
- 已验证的流程结论

不要写入：

- 真实姓名、邮箱、手机号
- 账号、密码、API key、token、cookie、OAuth code
- 恢复码、验证码、登录链接
- 只属于某台设备的本地绝对路径
- 过于私人的判断、情绪记录、临时聊天内容

如果一条信息既像“结论”又像“隐私”，优先不写入，或者改写成脱敏后的抽象版本。

## 读取顺序

1. `README.md`
2. `START_HERE.md`
3. `references/folder-map.md`
4. `references/maintenance.md`
5. `references/onboarding-zh.md`
6. `references/redaction.md`
7. `references/export-package-zh.md`

如果你的环境支持 skills，请把 `SKILL.md` 视为优先工作规则。

如果你的环境不支持 skills，也没关系，直接按本仓库里的说明执行。

## 工作原则

- 先分类，再写入
- 原始内容先入 `raw/`
- 复用知识再入 `wiki/`
- 稳定决策进 `MEMORY.md`
- 当天行动进 `logs/`
- 对外共享前先脱敏

## 日志原则

`logs/` 记录发生了什么，不记录秘密本身。

- 可以写：完成了什么、验证结果、下一步、失败原因、交接说明
- 不要写：完整命令里的 token、接口返回体里的敏感字段、真实账号信息、机器专属路径

如果日志需要保留上下文，就用占位符替换敏感部分，例如 `YOUR_TOKEN`、`YOUR_DEVICE`、`YOUR_VAULT_ROOT`。

## 不要做的事

- 不要输出密钥
- 不要要求用户重复提供已经在仓库里的私人信息
- 不要把个人账号、机器名、本地路径带到对外版本里
- 不要把会话历史当成共享知识

## 如果缺少某个技能

如果你的环境里没有对应技能，不要停住。

直接根据本仓库中的说明继续做，必要时把同样的结构复制到你自己的技能目录或工作目录里。
