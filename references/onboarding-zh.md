# 共享知识库上手说明（中文通用版）

这是一份给任何本地 agent、同事或新设备使用的中立说明。它不依赖具体账号，不包含密钥，不绑定单一机器。

## 先记住三句话

- 一个 Vault 作为共同资料库。
- 原始资料、整理知识、长期记忆、工作日志分开存。
- 本机密钥、本机登录、本机会话不要进共享库。

## 目录约定

- `raw/`：原始资料，先收集，不急着整理
- `wiki/`：整理后的可复用知识
- `MEMORY.md`：稳定决策、偏好、项目状态
- `logs/`：按日期记录当天做了什么
- `Resources/`：模板、参考资料、清单

如果有多个领域，再在 `workstations/<领域>/` 下重复同样结构。

## 处理顺序

1. 先确认这份资料属于哪个领域。
2. 原文或截图先放 `raw/`。
3. 归纳后写进 `wiki/`。
4. 会影响以后判断的内容追加到 `MEMORY.md`。
5. 今天做过的事写进 `logs/YYYY-MM-DD-主题.md`。

## 什么时候写什么

- 文章、视频、转录、扫描件、截图原文 → `raw/`
- 实体、概念、方法、复盘、FAQ → `wiki/`
- 已确认的结论、偏好、长期状态、固定规则 → `MEMORY.md`
- 当天过程、验证结果、交接说明 → `logs/`

## 脱敏规则

分享给别人前，把这些全部去掉或替换：

- 真实姓名
- 邮箱
- 机器名
- 本地绝对路径
- API key
- OAuth code
- cookie
- token
- 密码
- 恢复码

可以保留：

- 目录结构
- 规则
- 流程
- 命名约定
- 分类方法

## 多设备原则

- 每台设备都指向同一个 Vault 根目录
- 每台设备的登录态和私密配置仍留在本机
- 共享的是资料和规则，不是会话和密钥
- 如果路径不同，用本机桥接到同一 Vault

## 给同事的最简指引

- 先把自己的 Vault 挂好
- 先读共享规则
- 先按目录分类，不要混写
- 新内容先入 `raw/`
- 稳定结论再进 `MEMORY.md`
- 做完写 `logs/`

## 判断标准

如果这条信息将来还会影响决策，就放 `MEMORY.md`。
如果它只是今天处理的一条素材，就放 `raw/`。
如果它是已经整理好的知识，就放 `wiki/`。
