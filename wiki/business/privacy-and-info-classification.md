## 一、客户隐私协议（什么时候签、签什么）

### 触发时机
- **任何客户数据进入系统之前**（不是之后）
- 提供选股分析、持仓跟踪、定制策略等服务前
- 分享涉及客户持仓/成本的任何报告前

### 协议必须覆盖的内容

| 条款 | 说明 |
|------|------|
| 数据范围 | 明确收集什么：持仓、成本、交易记录、个人信息 |
| 使用目的 | 只用于本服务，不用于第三方分析或训练 AI |
| 存储方式 | 本地/加密云端，不存明文 token/密码 |
| 保留期限 | 服务结束后多久删除数据 |
| 转让限制 | 不出售、不转让给任何第三方 |
| 数据主权 | 客户随时可要求删除所有数据 |
| 法律适用 | 根据注册地选（香港法/内地法/新加坡法） |

### 最低可行版本（早期）
早期客户少，可用一份**简版书面确认函**代替正式合同：
```
本人确认已知晓 [你的产品名] 将收集我的 [持仓/成本] 信息，
仅用于提供选股分析服务，不会共享给第三方。
```
正式版找律师起草，费用约 HK$3000-8000。

## 二、知识点信息分级系统

### 标签规范（在 Obsidian 每条笔记顶部打标）

```
#公开 ✅     → 可发布/分享，无限制
#内部 🟡     → 团队内部，不对外
#客户专属 🔒 → 特定客户信息，严格保密
#草稿 ✏️     → 未审核，暂不流通
```

### 判断树（新增知识点时问自己）

```
这条信息涉及具体客户数据？
├─ 是 → #客户专属 🔒（必须隔离存储）
└─ 否 → 涉及公司内部流程/策略？
         ├─ 是 → #内部 🟡
         └─ 否 → 是否有原创竞争价值？
                  ├─ 是，不想公开 → #内部 🟡
                  └─ 否/愿意公开 → #公开 ✅
```

### 存储隔离原则
- `#客户专属` 笔记：只存本地，**永远不进 git / iCloud 共享 vault**
- `#内部` 笔记：可进私有 vault，不进公开仓库
- `#公开` 笔记：可进 shared-vault-workflow 或任何公开渠道

### 与 AI agent 协作边界
- 永远不把客户真实持仓/成本粘给 AI 对话（用占位符替代）
- AI 生成的分析结论可以是 `#公开`，但输入的客户数据不可以
- 在 HANDOFF.md 里用 `YOUR_HOLDINGS` 代替真实数字
